Ssl termination with pentaho

Feb 28, 2017 #docker #pentaho #proxy

This post builds on top off a previous post Setup an easy reverse proxy to the biserver

Problem

Its 2017!

They just gave the big red do not push button to the monkey, we are all fooked!

… unless …

Solution

We include ssl in our pentaho setup using ssl termination

Ingredients

  • a letsencrypt container
  • a modified haproxy based on the previously used dockercloud haproxy container
  • a randomly picked biserver
  • a docker-composition to glue these together:
biserver:
  image: serasoft/docker-pentaho
  environment:
    - FORCE_SSL=yes
    - VIRTUAL_HOST=*,https://*
  expose:
    - "8080"
  ports:
    - "8080"

proxy:
  image: interaction/haproxy:master
  links:
    - letsencrypt
    - biserver
  ports:
    - "80:80"
    - "443:443"
  # roles:
  #   - global
  volumes_from:
    - letsencrypt

letsencrypt:
  image: interaction/letsencrypt:master
  environment:
    - DOMAINS=mypublicdomain.com
    - EMAIL=noreply@mypublicdomain.com
    # - OPTIONS=--staging
  expose:
    - "80"

Start this composition with docker-compose

docker-compose up -d && docker-compose logs -f
